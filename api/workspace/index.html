    ValidationWorkspace

    The main user-facing API. Owns I/O providers and orchestrates the full prepare-then-compute workflow.

    Stateful analysis session for a single TRICERATOPS+ validation target.

    Construction is side-effect-free: the catalog query is deferred until the stellar field is first accessed (via .stars, .target, or .fetch_catalog()). Pass a pre-built stellar_field to skip the catalog query entirely.

    Only mission="TESS" is supported for prepared compute. Kepler/K2 support is experimental; passing a non-TESS mission will raise UnsupportedComputeModeError when compute_probs() is called.

    stars property

    All stars in the stellar field (target at index 0).

    target property

    The target star.

    stars_df property

    All stars in the stellar field as a pandas DataFrame.

    results property

    Most recent validation result, or None if not yet computed.

    fpp property

    False Positive Probability from the most recent run.

    nfpp property

    Nearby False Positive Probability from the most recent run.

    fetch_catalog()

    Explicitly fetch the stellar field from the catalog.

    Triggers the catalog query if not already loaded. Returns the StellarField. Subsequent calls return the cached field.

    This is the explicit alternative to the lazy-on-first-access pattern. Use it when you want to control exactly when the network call happens.

    add_star(star)

    Add a neighbor star to the stellar field. Invalidates cached results.

    Raises:

    Type Description
    ValueError

    If a star with the same TIC ID already exists.

    remove_star(tic_id)

    Remove a star by TIC ID. Invalidates cached results.

    Raises:

    Type Description
    ValueError

    If tic_id is the target star.
    ValueError

    If no star with tic_id is found.

    update_star(tic_id, **kwargs)

    Update fields on a star by TIC ID. Invalidates cached results.

    Accepts both direct Star attribute names and TIC-style aliases

    Teff -> stellar_params.teff_k mass -> stellar_params.mass_msun logg -> stellar_params.logg metallicity -> stellar_params.metallicity_dex

    Raises:

    Type Description
    ValueError

    If no star with tic_id is found.
    TypeError

    If an alias update is requested and stellar_params is None.
    AttributeError

    If an unknown attribute name is given.

    resolve_target(target)

    Resolve a target string (e.g. TOI number) using the injected resolver.

    Raises:

    Type Description
    RuntimeError

    If no ephemeris_resolver was provided.
    LightCurveError

    If resolution fails (propagated from resolver).

    set_resolved_target(target)

    Bind a resolved target to the workspace without performing I/O.

    This is intended for higher-level orchestration layers which have already resolved a target and need the workspace assembly path to use that result.

    prepare(light_curve, period_days, scenario_ids=None, external_lcs=None, contrast_curve=None, molusc_data=None)

    Assemble all I/O inputs and validate them for compute.

    This method performs all network and disk I/O: - TRILEGAL background population fetch (if not cached) - Any other provider-backed data assembly

    The returned PreparedValidationInputs is fully self-contained and can be inspected, serialized, or passed to compute_prepared().

    Returns:

    Type Description
    PreparedValidationInputs

    PreparedValidationInputs ready for engine.compute_prepared().

    Raises:

    Type Description
    UnsupportedComputeModeError

    If mission is not TESS.
    PreparedInputIncompleteError

    If stellar params are missing.
    ValidationInputError

    If light curve or period is invalid.

    compute_prepared(prepared)

    Run provider-free compute on pre-assembled inputs.

    No network or disk I/O occurs in this method.

    Parameters:

    Name Type Description Default
    prepared PreparedValidationInputs

    Output from prepare() or manually constructed.

    		 required

    Returns:

    Type Description
    ValidationResult

    ValidationResult, also stored internally for property access.

    compute_probs(light_curve, period_days, scenario_ids=None, external_lcs=None, contrast_curve=None, molusc_data=None)

    Run validation computation and cache the result.

    Convenience method that calls prepare() then compute_prepared(). Equivalent to::

    prepared = self.prepare(light_curve, period_days, ...)
result = self.compute_prepared(prepared)

    Returns:

    Type Description
    ValidationResult

    ValidationResult, also stored internally for property access.
